# ⚡ Checklist de Performance do Modelo Semântico | Power BI

Um checklist prático para garantir que seu modelo semântico esteja otimizado, performático e seguindo as melhores práticas de modelagem no Power BI.

---

## 📋 O que é?

Um guia de verificação focado em **performance e boas práticas de modelagem** para modelos semânticos no Power BI. Cada item foi pensado para identificar gargalos, otimizar consultas DAX e garantir que seu modelo escale de forma saudável.

---

## 🔍 O que cada item contém?

- **Pergunta-guia** → para reflexão antes de validar
- **Tarefa prática** → verificação objetiva e mensurável
- **Link de referência** → artigo ou documentação para aprofundar o conceito
- **Status com dropdown** → Concluído, Não Atende, Pendente ou N/A

---

## 📊 Funcionalidades do Excel

- ✅ **Dropdown funcional** na coluna STATUS — clique e selecione
- ✅ **Formatação condicional** — cores mudam automaticamente conforme o status
- ✅ **Contadores no cabeçalho** — acompanhe em tempo real quantos itens estão concluídos, pendentes ou não atendem
- ✅ **Links clicáveis** — cada item tem referência direta para estudo

---

## 🎯 Quando usar?

| Momento | Como usar |
|---------|-----------|
| **Após a modelagem** | Valide se o modelo está seguindo boas práticas |
| **Antes de publicar** | Garanta que a performance está adequada |
| **Troubleshooting** | Identifique gargalos em modelos lentos |
| **Code review** | Use como guia estruturado de revisão técnica |

---

## 📚 Referências utilizadas

O checklist é baseado em conceitos de:

- **SQLBI** — Boas práticas DAX, VertiPaq, modelagem star schema
- **Microsoft Learn** — Documentação oficial Power BI e Analysis Services
- **DAX Patterns** — Padrões otimizados de cálculo
- **Marco Russo & Alberto Ferrari** — The Definitive Guide to DAX
